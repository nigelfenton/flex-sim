# flex-sim

A **synthetic FlexRadio-6000 emulator** for testing [AetherSDR](https://github.com/aethersdr/AetherSDR) — a deterministic, hardware-free spectrum/waterfall/CW test bench.

flex-sim impersonates a FlexRadio 6000's data plane: it answers SmartSDR discovery and the FlexLib control handshake, then streams synthetic **VITA-49** panadapter / waterfall / meter data so AetherSDR renders a waterfall, S-meter and TX meters from a programmable signal engine. No radio required.

It serves a **live browser control panel** with a pattern picker (each pattern shows *what it exercises in AE*), dBm/S-unit level controls, TX keying, and CW — including driving AE's own CWX keyer.

- Pure **Python 3.8+ stdlib** — zero dependencies.
- Drives panadapter, waterfall, **S-meter** (dBm/S-unit), **TX power/SWR meters**, **CW + CWX**.
- Patterns for the known AE waterfall issues (auto-black #3586, TX-blank #2126/#1916, freq mapping #3457, …).

> Status: **v0.1 (beta).** Wire format reverse-engineered from AE's own decoder — see [`PROTOCOL.md`](PROTOCOL.md). Design notes in [`DESIGN.md`](DESIGN.md).

## ⚠️ Networking (read this first)

flex-sim **and** AetherSDR both bind UDP **`:4992`**, so flex-sim needs its **own network stack, separate from AE's**. You **cannot** run it natively on the same host as AE — they fight over `:4992`. Pick one:

- **Same physical machine as AE, on Windows → use WSL2.** WSL2 gives flex-sim its own IP (`172.x`), reachable from the Windows host, with no `:4992` clash. Put `[wsl2]\nlocalhostForwarding=false` in `%USERPROFILE%\.wslconfig` (so WSL's `:4992` doesn't relay onto the host), then run flex-sim **inside WSL** and let AE discover it. *This is the proven same-machine path on Windows.*
- **Separate box on the LAN** (Pi / NUC / spare PC / VM) — easiest; run the `.exe`, Python, or Docker there and AE auto-discovers it.
- **`--ae <AE-ip>`** — flex-sim also unicasts discovery straight to AE (helps across subnets / when broadcast doesn't traverse).

**Docker caveat:** on **Linux**, a `macvlan` network gives the container its own LAN IP (clean — see `docker-compose.yml`). On **Docker Desktop for Windows/Mac**, containers aren't reachable at their own IP from the host and `macvlan` isn't supported, so **Docker does *not* solve the same-host case there — use WSL2.** Docker is for a **separate Linux box**.

## Run it

### 1. Python (any OS, no install)
```bash
python3 flex_sim.py --ae <AE-ip>          # unicast discovery to AE as well as broadcast
# then open the control panel:            http://<this-host-ip>:8731/
```
Common flags: `--pattern carrier` · `--bins 1024` · `--fps 20` · `--ctl-port 8731` · `--version` · `-h`.

### 2. Docker
```bash
docker build -t flex-sim .

# Simplest — on a machine that is NOT the AE host:
docker run --rm --network host flex-sim --ae <AE-ip>

# Same LAN, own identity (recommended) — give the container its own IP via macvlan
# (edit parent iface / subnet in docker-compose.yml), then:
docker compose up
```
Control panel: `http://<container-or-host-ip>:8731/`.

### 3. Standalone executable (no Python)
A single-file binary is produced by PyInstaller (see [`.github/workflows/build.yml`](.github/workflows/build.yml) for the release build matrix, or build locally):
```bash
pip install pyinstaller
pyinstaller --onefile --name flex-sim flex_sim.py
./dist/flex-sim --ae <AE-ip>              # dist/flex-sim.exe on Windows
```

## Pointing AetherSDR at it
With flex-sim running, AE should list it in the radio chooser (model `FLEX-6600`, serial `EMULATE01`) — select and connect. If it doesn't appear, confirm the networking rule above and that you passed `--ae <AE-ip>`.

## Control panel
`http://<ip>:8731/` — pick a pattern (the **hint box** says what to look for in AE), set the noise floor / signal level in **dBm (with S-unit)**, signal width, noise colour; key **TX** (forward-power + SWR meters); send **CW** (normal / full break-in). The authentic CW path is driven from AE's own CWX window — flex-sim follows AE's WPM/QSK.

## Patterns
`noise_floor` · `ramp` · `cal_tones` · `carrier` · `cw` · `swept_carrier` · `comb` · `step` · `impulse` · `staircase` · `noise` · `tx_blank`. See the panel hint box or the module docstring for what each one tests.
